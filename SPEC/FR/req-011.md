# REQ-011: Enhanced Search — Search Panel, Menu Search, Docs Search

## Summary

KafGraph shall provide a comprehensive search experience that goes beyond the
current graph-browser node search, adding a dedicated search results panel,
cross-view menu navigation search, and full-text documentation search.

## Motivation

The current search is limited to the graph-browser view and renders results only
as a Cytoscape graph visualization. Users need structured search results they can
scan quickly, the ability to navigate the UI itself via search, and the ability
to search documentation content without leaving the management UI.

## Requirements

### Search Results Panel (Phase 1)

- **REQ-011.1**: A dedicated search results panel shall display matching nodes
  and edges in a sortable, filterable table with columns: ID, Label, Key
  Properties, Created At.
- **REQ-011.2**: Each result row shall be clickable, navigating to the
  graph-browser centered on that node.
- **REQ-011.3**: The results panel shall support pagination (default 25 per page)
  and a total count indicator.
- **REQ-011.4**: Results shall highlight the matched terms within property values.

### Menu / Navigation Search (Phase 2)

- **REQ-011.5**: The global search input shall support a `/` prefix to switch
  into navigation mode, matching against view names, menu items, and actions
  (e.g., `/config` → Configuration view, `/compliance` → Compliance view).
- **REQ-011.6**: Navigation results shall appear as a dropdown overlay beneath
  the search input, selectable via keyboard (arrow keys + Enter).

### Documentation Search (Phase 3)

- **REQ-011.7**: A docs search mode (triggered by `?` prefix) shall perform
  full-text search across documentation content served by the Jekyll site.
- **REQ-011.8**: Documentation results shall display title, excerpt with
  highlighted matches, and a link to the full page.

## Dependencies

- REQ-009 (Management UI) provides the shell, sidebar, and global search input.
- Graph search API (`/api/v2/graph/search`) already supports the backend query.
