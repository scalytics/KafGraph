# REQ-001: Makefile-driven workflow

**Status**: Implemented
**Phase**: 0
**Priority**: Must
**Source**: Development setup plan

## Description

All build, test, lint, coverage, Docker, and documentation tasks must be driven
through a single Makefile with self-documenting `make help` output. The Makefile
must support at least 40 targets organized into logical groups.

## Acceptance Criteria

- [x] `make help` lists all targets with descriptions
- [x] `make build` compiles the binary with ldflags
- [x] `make test` runs all unit tests
- [x] `make lint` runs golangci-lint
- [x] `make ci` runs the full CI pipeline locally
- [x] `make release-check` validates all release gates
