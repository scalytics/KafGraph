# REQ-009: KafClaw roster evolution ŌĆö emit updated rosters on skill set changes

**Status**: Implemented
**Phase**: 4
**Priority**: Should
**Source**: Group Inspector feature requirement

## Description

KafClaw agents currently emit a single `roster` envelope per session, declaring
their skill set at startup. There is no mechanism to track when an agent gains or
loses a skill over time.

Agents MUST emit an updated `roster` envelope to `group.<name>.control.roster`
whenever their skill set changes (skill added, removed, or version updated). This
enables KafGraph to build a timeline of capability evolution per agent.

## Envelope Extension

The `RosterPayload` is extended with two optional fields:

| Field     | Type   | Description |
|-----------|--------|-------------|
| `Version` | int    | Monotonically increasing roster version counter |
| `Action`  | string | `"full"` (complete roster), `"add"` (skills added), `"remove"` (skills removed) |

Backward compatibility: existing rosters without `Version`/`Action` are treated as
`Action: "full"` with `Version: 0`.

## Agent Requirements

1. On startup, emit a `roster` envelope with `Action: "full"` and `Version: 1`.
2. When a skill is added at runtime, emit with `Action: "add"`, incremented `Version`,
   and only the newly added skills in the `Skills` array.
3. When a skill is removed, emit with `Action: "remove"`, incremented `Version`, and
   the removed skills in the `Skills` array.
4. The envelope `Timestamp` serves as the authoritative time for when the change occurred.

## KafGraph Ingest Behavior

- `Action: "full"` or absent: create/update `HAS_SKILL` edges with `declaredAt` and
  `rosterVersion` properties.
- `Action: "add"`: same as full for the listed skills.
- `Action: "remove"`: set `removedAt` property on the `HAS_SKILL` edge (soft delete).
  The edge is preserved for historical queries but excluded from active skill counts.

## Per-Agent Skill Inspection

The `GET /api/v2/mgmt/skills/by-agent` endpoint supports:

- **`?agent=name`** ŌĆö Filter to a single agent (case-insensitive). Returns only that
  agent's data, enabling per-agent drill-down in the UI.
- **`skillHistory`** ŌĆö Each agent entry includes a `skillHistory` array derived from
  `HAS_SKILL` edge properties. Each event contains:
  - `skill` ŌĆö skill name
  - `declaredAt` ŌĆö RFC 3339 timestamp when the skill was declared
  - `rosterVersion` ŌĆö roster version counter at the time of the event
  - `removedAt` ŌĆö RFC 3339 timestamp if the skill was removed (empty if active)
  - `active` ŌĆö boolean, `false` when `removedAt` is set

Example response for a single agent:
```json
{
  "agents": [{
    "id": "n:Agent:formatter", "name": "formatter",
    "declaredSkills": ["ascii_doc", "format_html", "proofread"],
    "skillUsage": [...],
    "skillHistory": [
      {"skill": "ascii_doc",    "declaredAt": "2026-03-01T09:00:06Z", "rosterVersion": 1, "active": true},
      {"skill": "proofread",    "declaredAt": "2026-03-01T09:00:06Z", "rosterVersion": 1, "active": true},
      {"skill": "format_html",  "declaredAt": "2026-03-01T09:02:18Z", "rosterVersion": 2, "active": true}
    ]
  }],
  "skillRoster": [...]
}
```

## UI: Skill Evolution History

The Group Inspector view includes:

1. **Agent filter** ŌĆö Dropdown selector to inspect a single agent's skills and history
2. **Skill Evolution History table** ŌĆö Chronological list of all roster events across
   agents, showing time, agent, skill, version, and active/removed status
3. **Roster Events metric** ŌĆö Summary card counting total roster evolution events
